## 📖 Descrição

Este documento apresenta o estudo sobre requisitos de software, abordando seus tipos, a importância dentro do desenvolvimento de sistemas e o processo de Engenharia de Requisitos aplicado na prática.

Requisitos de software são descrições dos serviços, funcionalidades e restrições que um sistema deve oferecer. Eles são levantados nas fases iniciais do desenvolvimento e representam as necessidades dos usuários traduzidas para uma linguagem técnica compreensível pelos desenvolvedores.

---

## 🎯 Tipos de Requisitos

### 🧩 Requisitos Funcionais (RF)
Descrevem as funcionalidades que o sistema deve executar, ou seja, o que ele deve fazer.

**Exemplo:**
> O sistema deve permitir que o usuário pesquise livros por título ou autor.

---

### ⚙️ Requisitos Não Funcionais (RNF)
Definem características de qualidade e restrições do sistema, como desempenho, segurança e usabilidade.

**Exemplo:**
> O sistema deve responder às pesquisas em até 3 segundos.

---

### 🏛️ Requisitos de Domínio
São específicos do contexto onde o sistema será aplicado, refletindo regras da área de atuação.

**Exemplo:**
> Em um sistema de biblioteca, o controle de empréstimos é um requisito típico do domínio.

---

## 🔄 Engenharia de Requisitos

A Engenharia de Requisitos é o processo responsável por descobrir, organizar, documentar e gerenciar os requisitos de um sistema.

### A. Elicitação
Etapa de levantamento dos requisitos junto aos stakeholders.

**Como aplicar:**
- Entrevistas
- Questionários
- Observação
- Brainstorming
- Prototipação

---

### B. Análise
Organização dos requisitos e resolução de conflitos.

**Como aplicar:**
- Priorização de funcionalidades  
- Eliminação de ambiguidades  
- Comparação de necessidades  

---

### C. Especificação
Documentação estruturada dos requisitos.

**Como aplicar:**
- Modelo tradicional: documento SRS  
- Métodos ágeis: histórias de usuário  

**Exemplo de história de usuário:**
> Como usuário, quero pesquisar livros para encontrar rapidamente o que preciso.

---

### D. Validação
Verificação se os requisitos atendem às necessidades reais.

**Como aplicar:**
- Revisões técnicas  
- Reuniões com stakeholders  
- Testes com protótipos  

---

### E. Gestão de Requisitos
Controle das mudanças ao longo do projeto.

**Como aplicar:**
- Uso de ferramentas como Jira ou Trello  
- Acompanhamento do status  
- Avaliação de impacto de mudanças  

---

## 👥 Stakeholders

Stakeholders são todas as pessoas ou organizações interessadas no sistema, como:

- Clientes  
- Usuários finais  
- Desenvolvedores  
- Gerentes  
- Órgãos reguladores  

---

## ✅ Boas Práticas para Requisitos

Um requisito de qualidade deve ser:

- **Claro** → sem ambiguidades  
- **Completo** → com todas as informações necessárias  
- **Verificável** → possível de testar  
- **Consistente** → sem conflitos com outros requisitos  

---

## 💡 Aprendizados

O estudo mostrou que a definição correta dos requisitos é essencial para o sucesso do sistema, pois erros nessa fase podem gerar retrabalho e problemas ao longo do desenvolvimento.

A Engenharia de Requisitos permite organizar melhor as necessidades dos usuários e garantir que o sistema atenda às expectativas de forma eficiente.
