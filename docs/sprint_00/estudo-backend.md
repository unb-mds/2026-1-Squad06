# Estudo sobre tecnologias de back-end

## Contexto
O back-end é uma das partes mais cruciais de um projeto, sendo responsável pelo processamento de dados, segurança e pelo funcionamento principal da aplicação.
O estudo do sprint 00 teve como objetivo pesquisar sobre as principais tecnologias de back-end utilizadas no mercado e descobrir uma ideal para o projeto.

## Tecnologias pesquisadas

### Python
- **Django**: Framework muito seguro com excelente painel de admin, voltado para full-stacks e sites complexos
- **FastAPI**: Voltado para APIs modernas, valida dados automaticamente, ideal para machine learning
- **Flask**: Framework minimalista, bom para protótipos, microserviços e projetos simples

### JavaScript
- **Express.js**: Framework mais popular, padrão da indústria para APIs, pequena curva de aprendizado
- **NestJS**: Usa TypeScript, arquitetura modular, ideal para aplicações de larga escala

### Java
- **Spring Boot**: Padrão para microserviços, ecossistema completo
- **Quarkus**: Otimizado para nuvem, inicialização rápida e baixo consumo de memória

### C#
- **ASP.NET**: Multiplataforma, open source, excelente para APIs
- **ASP.NET MVC**: Tradicional, usado em sistemas antigos
- **Nancy**: Leve e minimalista, fácil de aprender
- **Blazor**: Permite usar C# no backend e frontend, alternativa ao JavaScript
